Implement code changes to address GitHub issue requirements.

Input: issue_number

Steps:
1. Fetch issue: gh issue view <number> --json title,body
2. Read implementation plan from issue comments
3. Explore codebase to understand current structure
4. Implement changes layer-by-layer:
   - Database: Create migrations if needed, update models
   - Backend: Update services, routers, schemas per plan
   - Frontend: Update components, API client, UI per plan
   - Tests: Update or create tests for changes
5. Track modified files and line changes
6. Report implementation summary:
   - Files modified (with line counts)
   - What was implemented
   - Any blocking issues

Implementation approach:
- Follow plan from planning phase exactly
- Make specific file changes with exact line numbers
- Keep changes focused and minimal
- Update tests as needed
- Report what changed and why

Output format:
- Files modified:
  - backend/app/schemas.py: +5 lines
  - backend/app/services/chore_service.py: +10 lines
  - etc.
- Implementation summary: [describe what was done]
- Status: Ready for testing OR Blocked due to [issue]
